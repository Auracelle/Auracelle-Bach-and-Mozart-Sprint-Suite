# Quick Start Guide - Auracelle Sprint Suite

## Get Your Sprint Suite Live in 5 Minutes!

### Step 1: Create Repository
1. Go to [github.com](https://github.com) and log in
2. Click the green "New" button
3. Name it: `Auracelle-Sprint-Suite`
4. Select "Public" (or "Private" if you prefer)
5. Do NOT check "Add a README file"
6. Click "Create repository"

### Step 2: Upload Files
1. Click "uploading an existing file"
2. Drag and drop all files:
   - `index.html` (the Sprint Suite itself)
   - `README.md`
   - `LICENSE`
   - `DEPLOYMENT.md`
   - `QUICKSTART.md`
   - `COLLABORATION.md`
3. Add commit message: "Initial Sprint Suite upload - 109 Sprints"
4. Click "Commit changes"

### Step 3: Enable GitHub Pages
1. Click "Settings" in your repository
2. Click "Pages" in the left menu
3. Under "Source," select "Deploy from a branch"
4. Under "Branch," select "main"
5. Under folder, select "/ (root)"
6. Click "Save"
7. Wait 1-2 minutes and refresh

Your site will be at:
`https://YOUR-USERNAME.github.io/Auracelle-Sprint-Suite/`

## Updating Your Sprint Suite

### Add New Sprints
1. Go to your repository
2. Click on `index.html`
3. Click pencil icon (edit)
4. Find the `sprints` array (around line 500-2000)
5. Add your new sprint following the format
6. Commit changes
7. Live in 1-2 minutes!

### Modify Existing Sprints
1. Edit `index.html` same way
2. Find the sprint you want to change
3. Update text or details
4. Commit with clear message
5. Changes go live automatically!

## Troubleshooting

**Site not loading?**
- Wait full 2-3 minutes after enabling Pages
- Check Settings â†’ Pages shows green "Your site is live"
- Try opening in incognito/private browser window

**Data not saving?**
- Check browser allows localStorage
- Not in incognito mode (data won't persist)
- Try different browser

**Can't add collaborators?**
- Verify you're the repository owner
- Check they have GitHub account
- Make sure you're in Settings, not somewhere else
